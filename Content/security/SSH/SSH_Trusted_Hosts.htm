SSH Trusted Hosts

The first time you successfully log on to and are authenticated by an SSH server, that server's information is stored as a trusted host. Each subsequent time you log on to that server, the server information sent is compared to the information stored as a trusted host. If the information does not match, WS_FTP alerts you to the potential security issue so that you can further verify that the server connection is secure.  

Note: Trusted Hosts are sometimes used as a method of authenticating an SSH connection; however, WS_FTP does not support that authentication method.

To remove a trusted host:

  1. From the main window, select Tools > Options. The Program Options dialog appears. 

  2. Select SSH > Trusted Hosts.

  3. In the Trusted Hosts list, locate and click the server you wish to remove.

  4. Click Remove.

To export a trusted host:

  1. From the main window, select Tools > Options. The Program Options dialog appears. 

  2. Select SSH > Trusted Hosts.

  3. In the Trusted Hosts list, locate and click the server you wish to export.

  4. Click Export. Follow the on-screen prompts to complete the process.
