Client Certificate Verification

If the server you are attempting to make a connection to requires your client to send a certificate back to the server, you must:

  1. Configure the site with the FTP/SSL or FTP/Implicit SSL server type.

  2. Create a certificate.

  3. Send the certificate signing request file to your server administrator.

  4. Once the server administrator signs your certificate signing request, a certificate file will be sent back to you.

  5. When you receive the file, you need to import the signed certificate file.

    1. From Program Options, select SSL > Client Certificates.

    2. Click the Import button to start the Import Wizard.

    3. When prompted for the Certificate File, provide the path to the signed certificate file you received from the server administrator.

    4. When prompted for the Private Key File, provide the path to the private key file of the certificate that you originally created.

    5. When prompted for the Certificate Name, provide a new name for the certificate that you created that signifies that the certificate has been signed by the server administrator.

  6. Select the newly imported certificate in Client Certificate drop down on the site's SSL page.

    1. Select the site in the Site Manager.

    2. Click the Edit button.

    3. Select Advanced > SSL.

    4. Select your signed certificate in the Client Certificate list.

    5. Click Ok.

  7. Connect to the server.
